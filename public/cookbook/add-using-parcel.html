Issue

We need to add Scrawl-canvas functionality to a HTML5/CSS/JS project that uses the Parcel web application bundler.

Solution

Include Scrawl-canvas in the project using Yarn or NPM, then import it into your Javascript code as normal:

index.html

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset='utf-8'>
    <meta name='viewport' content='width=device-width,initial-scale=1'>

    <title>Your project name</title>
</head>

<body>
    <canvas id="mycanvas"></canvas>

    <script src="./index.js"></script>
</body>
</html>

index.js

import scrawl from 'scrawl-canvas';

// Your Javascript code can now include Scrawl-canvas code

Discussion

Scrawl-canvas is entirely modular. However when including the Scrawl-canvas package into another project using NPM or Yarn, the library's package.json manifest tells the project's toolchain to use the minified, single file version of the library.

We use rollup.js to generate that minified file and, as part of its functionality, rollup.js automatically handles (and future-proofs) module handling.

The outcome of all this is that Scrawl-canvas should — in theory — play nicely with other code bundlers such as Parcel. It also means that the import scrawl from 'scrawl-canvas' code can be used in any <script> tag — there should be no need to explicitly mark the tag as type="module".

Note that testing with bundlers has (to date) been limited. Your mileage may vary.
